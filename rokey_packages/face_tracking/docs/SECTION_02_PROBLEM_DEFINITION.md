##  2.3 왜 어려운가? (기술적 챌린지)

### 핵심 난제

```
┌─────────────────────────────────────────────────────────────────┐
│                    로봇팔 초병 시스템의 기술적 난제               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   실시간성 (Real-time)                                        │
│  ├─ 사람 움직임: 예측 불가, 급격한 방향 전환                     │
│  ├─ 요구 반응 속도: <100ms (인간 반응 한계 이하)                 │
│  └─ 파이프라인 지연: 카메라 → AI → 로봇 전 구간 최적화 필요      │
│                                                                 │
│   권총 파지 (Gun Gripping)                                    │
│  ├─ SAM3 권총: 비표준 형상, 그리퍼 파지 난이도 높음             │
│  ├─ 파지력 제어: 너무 약하면 낙하, 너무 강하면 손상              │
│  └─ 정렬 정밀도: 총구 방향과 카메라 시야 정렬 필요               │
│                                                                 │
│   안정성 (Stability)                                          │
│  ├─ 로봇팔 진동: 고속 이동 시 오버슈트/진동 발생                 │
│  ├─ 추적 연속성: 얼굴 가림, 검출 실패 시에도 추적 유지           │
│  └─ 24시간 운용: 장시간 안정적 동작 보장                         │
│                                                                 │
│   통합 (Integration)                                          │
│  ├─ 다중 시스템 연동: 비전 + 로봇 + 음성 + 웹 동기화             │
│  ├─ 상태 전이 관리: 복잡한 시나리오 분기 처리                     │
│  └─ 안전 시스템: 오발 방지, 비상 정지 등                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 기술별 난이도 분석

| 기술 영역 | 난제 | 난이도 | 해결 방안 |
|----------|------|--------|----------|
| **권총 파지** | SAM3 비표준 형상 파지 |  높음 | OnRobot RG2 + 커스텀 지그 |
| **얼굴 검출** | 실시간 + 정확도 동시 확보 |  높음 | YOLOv8 + TensorRT FP16 |
| **상태 추정** | 노이즈 속 위치/속도 추정 |  높음 | 9-state EKF |
| **로봇 제어** | 빠르고 부드러운 추적 |  높음 | Joint-space 직접 제어 |
| **피아식별** | 빠른 분류 + 높은 정확도 |  중간 | CNN 분류기 |
| **음성 인식** | 소음 환경 암구호 인식 |  중간 | Whisper STT |
| **시스템 통합** | 실시간 동기화 |  중간 | ROS2 Topic/Service |

##  2.8 실제 구현한 해결책

### 핵심 문제 → 해결 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                    문제점 → 구현된 해결책                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제 1: 처리 속도 병목 (2~4 Hz)                              │
│  ──────────────────────────────────                             │
│  원인: MoveIt IK 계산 (50ms+), 좌표 변환 지연                   │
│                                                                 │
│   해결: Joint-space 직접 제어                                 │
│  • IK 계산 제거 → 0.5ms 이하로 단축                            │
│  • J1(수평) + J4(수직) 2축만 사용                               │
│  • 결과: 2Hz → 50Hz (25배 향상)                                │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제 2: 노이즈로 인한 떨림/불안정                           │
│  ──────────────────────────────────────                         │
│  원인: Depth 센서 노이즈, 검출 불안정, 지연 누적                │
│                                                                 │
│   해결: 9-state EKF (Extended Kalman Filter)                  │
│  • 상태 벡터: [x, y, z, vx, vy, vz, ax, ay, az]                │
│  • 위치 + 속도 + 가속도 동시 추정                              │
│  • 노이즈 필터링 + 미래 위치 예측                              │
│  • 결과: 떨림 70% 감소, 추적 안정성 대폭 향상                  │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제 3: YOLO 추론 속도 부족                                 │
│  ────────────────────────────────                               │
│  원인: CPU 추론, 원본 모델 무거움                               │
│                                                                 │
│   해결: TensorRT FP16 최적화                                  │
│  • YOLOv8-face 모델 TensorRT 변환                              │
│  • FP32 → FP16 정밀도 최적화                                   │
│  • 결과: ~50ms → ~5ms (10배 향상)                              │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제 4: 얼굴 소실 시 추적 실패                              │
│  ────────────────────────────────────                           │
│  원인: 검출 실패 시 즉시 추적 종료                              │
│                                                                 │
│   해결: EKF 예측 기반 지속 추적                               │
│  • 검출 실패해도 속도 기반 위치 예측                           │
│  • 3초간 예측 추적 유지                                        │
│  • 3초 후 홈 위치 복귀                                         │
│  • 결과: 일시적 가림에도 연속 추적 유지                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### EKF 9-state 설계 상세

```
상태 벡터 (9개 상태 변수):
──────────────────────────
         ┌   x   ┐   위치 (m)
         │   y   │
         │   z   │
    X =  │  vx   │   속도 (m/s)
         │  vy   │
         │  vz   │
         │  ax   │   가속도 (m/s²)
         │  ay   │
         └  az   ┘

상태 전이 모델 (등가속도 모델):
──────────────────────────────
    x(k+1) = x(k) + vx(k)·dt + 0.5·ax(k)·dt²
    vx(k+1) = vx(k) + ax(k)·dt
    ax(k+1) = ax(k)  (가속도 유지 가정)

관측 모델:
──────────
    z = [x, y, z]  (카메라에서 측정된 3D 위치)

예측 (Predict):
──────────────
    X̂(k|k-1) = F · X̂(k-1|k-1)
    P(k|k-1) = F · P(k-1|k-1) · F^T + Q

갱신 (Update):
─────────────
    K = P(k|k-1) · H^T · (H · P(k|k-1) · H^T + R)^(-1)
    X̂(k|k) = X̂(k|k-1) + K · (z - H · X̂(k|k-1))
    P(k|k) = (I - K · H) · P(k|k-1)
```
