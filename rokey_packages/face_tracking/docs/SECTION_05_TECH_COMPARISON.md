##  5.2 상태 추정 (State Estimation)

### 후보 기술 비교

| 기술 | 노이즈 제거 | 속도 추정 | 계산 비용 | 복잡도 |
|------|------------|----------|----------|--------|
| 이동 평균 (MA) | ⭐⭐ |  | 매우 낮음 | ⭐ |
| 저역통과 필터 (LPF) | ⭐⭐⭐ |  | 낮음 | ⭐ |
| Kalman Filter | ⭐⭐⭐⭐ |  | 낮음 | ⭐⭐ |
| **Extended KF (EKF)** | **⭐⭐⭐⭐⭐** |  | 낮음 | ⭐⭐⭐ |
| Particle Filter | ⭐⭐⭐⭐⭐ |  | 높음 | ⭐⭐⭐⭐ |
| UKF | ⭐⭐⭐⭐⭐ |  | 중간 | ⭐⭐⭐⭐ |

### 선택: 9-state EKF

```
┌─────────────────────────────────────────────────────────────────┐
│                     선택 근거: 9-state EKF                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  상태 벡터: x = [x, y, z, vx, vy, vz, ax, ay, az]ᵀ              │
│                 └─위치─┘  └─속도─┘   └─가속도─┘                 │
│                                                                 │
│   선택 이유                                                    │
│  ─────────────                                                  │
│  1. 노이즈 제거: RealSense Depth 노이즈 (±10mm) 효과적 제거     │
│  2. 속도 추정: 측정 없이 속도/가속도 추정 가능                   │
│  3. 예측 가능: 미래 위치 예측 → 지연 보상                        │
│  4. 계산 효율: ~0.15ms (실시간 충분)                             │
│                                                                 │
│   탈락 이유                                                    │
│  ─────────────                                                  │
│  • 이동평균/LPF: 속도 추정 불가, 지연 발생                       │
│  • Particle Filter: 계산 비용 과다 (수백 파티클)                 │
│  • UKF: 비선형 불필요 (우리 문제는 선형에 가까움)                │
│                                                                 │
│   최종 성능                                                    │
│  ─────────────                                                  │
│  • 처리 시간: ~0.15ms                                           │
│  • 노이즈 감소: 70% (±10mm → ±3mm)                              │
│  • 지연 보상: ~33ms 예측 가능                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

##  5.3 로봇 제어 (Robot Control)

### 후보 기술 비교

| 기술 | 계산 시간 | 제약 처리 | 부드러움 | 복잡도 |
|------|----------|----------|----------|--------|
| P/PD Control | ~0.1ms |  | ⭐⭐ | ⭐ |
| PID Control | ~0.1ms |  | ⭐⭐⭐ | ⭐ |
| **P-Control (Joint)** | **~0.5ms** |  | ⭐⭐⭐ | ⭐ |
| MPC (Cartesian) | ~50ms |  | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| DMP | ~5ms | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

### 선택: Joint-space P-Control

```
┌─────────────────────────────────────────────────────────────────┐
│               제어 방식 비교: Cartesian vs Joint                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Cartesian Space (MPC)          Joint Space (P-Control)    │
│  ────────────────────────         ──────────────────────────    │
│                                                                 │
│  [3D 좌표] → [IK 계산] → [관절]   [3D 좌표] → [구면좌표] → [J1,J4]│
│               ↑                                  ↑               │
│           ~50ms 지연                         ~0.5ms 지연         │
│                                                                 │
│  문제점:                          장점:                          │
│  • IK 필요 (계산 비용 높음)       • IK 불필요                    │
│  • 특이점 위험                    • 특이점 없음                  │
│  • 6관절 모두 움직임              • 2관절만 사용 (J1+J4)         │
│  • 복잡한 튜닝                    • 단순한 파라미터              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                  핵심 발상 전환                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  기존 사고: "로봇 끝을 얼굴 위치로 이동시켜야 한다"              │
│           → 6축 IK 필요 → 복잡 → 느림                           │
│                                                                 │
│  새로운 사고: "로봇이 얼굴 방향만 바라보면 된다"                 │
│           → 2축 (수평+수직) → 단순 → 빠름                       │
│                                                                 │
│  결과:                                                          │
│  • J1: 수평 방향 (베이스 회전)                                  │
│  • J4: 수직 방향 (손목 피치)                                    │
│  • 나머지 (J2,J3,J5,J6): 고정                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 성능 비교

| 항목 | MPC (Cartesian) | P-Control (Joint) | 개선 |
|------|-----------------|-------------------|------|
| 계산 시간 | ~50ms | **~0.5ms** | **100배** |
| 제어 주파수 | 20Hz | **50Hz** | **2.5배** |
| IK 필요 |  필요 |  불필요 | - |
| 특이점 |  위험 |  없음 | - |
| 구현 난이도 | 높음 | **낮음** | - |

##  5.7 Depth 노이즈 처리

### 후보 기술 비교

| 기술 | 이상치 제거 | 속도 | 복잡도 |
|------|------------|------|--------|
| Mean |  | O(n) | ⭐ |
| **Trimmed Mean** |  | O(n log n) | ⭐⭐ |
| Median |  | O(n log n) | ⭐⭐ |
| MAD |  | O(n log n) | ⭐⭐⭐ |

### 선택: Trimmed Mean

```
┌─────────────────────────────────────────────────────────────────┐
│                     선택 근거: Trimmed Mean                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  예시 데이터: [580, 585, 590, 595, 1200]  (1200은 이상치)       │
│                                                                 │
│  • Mean:         790  ← 이상치에 취약                           │
│  • Median:       590  ← 정확하지만 정보 손실                    │
│  • Trimmed Mean: 587  ← 이상치 제거 + 정보 보존                 │
│                                                                 │
│  구현:                                                          │
│  ──────                                                         │
│  1. 데이터 정렬                                                 │
│  2. 상하위 20% 제거                                             │
│  3. 나머지 평균                                                 │
│                                                                 │
│  장점:                                                          │
│  ──────                                                         │
│  • Median보다 정보 보존                                         │
│  • Mean보다 강건함                                              │
│  • RealSense Depth 노이즈에 최적                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
