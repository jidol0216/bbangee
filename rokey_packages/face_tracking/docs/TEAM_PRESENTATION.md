# 4️⃣ 헤드샷 트래킹 (태슬라)

> ✅ **이 섹션은 PRESENTATION.md 내용을 기반으로 작성됨**

## 4.1 문제 정의

### 얼굴 추적이 왜 어려운가?

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   얼굴 감지   │ ──▶ │  3D 위치 추정  │ ──▶ │   로봇 제어   │
│  (2D 이미지)  │     │  (깊이 정보)   │     │ (6축 매니퓰레이터)│
└──────────────┘     └──────────────┘     └──────────────┘
       ↓                    ↓                    ↓
   🔴 노이즈 多          🔴 좌표계 복잡         🔴 특이점 문제
   🔴 조명 변화          🔴 깊이 오차           🔴 IK 계산 복잡
   🔴 측면 얼굴          🔴 동기화 문제         🔴 반응 지연
```

### 우리가 풀고자 한 문제

1. **빠른 감지**: 실시간 30fps 이상
2. **정확한 추정**: 3D 위치 오차 최소화
3. **부드러운 제어**: 떨림 없는 로봇 동작
4. **안전한 동작**: 특이점/충돌 방지

## 4.3 MPC → Joint-space 전환

### 🚨 MPC 제어기의 문제점

```
MPC 파이프라인:
  목표 위치 (x,y,z) → IK 계산 → 관절각 (θ1...θ6) → 로봇 이동
                         ↓
                    ⏱️ 계산 시간 병목
                    ⚠️ 특이점 위험
```

### 💡 발상의 전환

```
┌─────────────────────────────────────────────────────────────────┐
│ 기존 MPC 접근 (위치 추적)                                        │
│   "로봇 End-Effector를 얼굴 위치(x,y,z)로 이동시켜라"            │
│   → 6축 IK 계산 필요 → 특이점 위험 → 복잡함                      │
└─────────────────────────────────────────────────────────────────┘
                                ▼
                          🔄 발상 전환!
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│ 새로운 Joint 접근 (방향 추적)                                    │
│   "로봇이 얼굴 방향만 바라보면 된다!"                             │
│   → 2축만 제어 (J1: 수평, J4: 수직)                             │
│   → IK 불필요 → 특이점 없음 → 단순함                            │
└─────────────────────────────────────────────────────────────────┘
```

```
│                     Face Tracking Node                               │
│      Depth Extraction → TF2 Transform → EKF 9-state → Marker        │
│              EKF: ~0.15ms, TF2+EKF: ~2ms                            │
└───────────────────────────────┬─────────────────────────────────────┘
                                │ /face_tracking/marker_robot
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                    Joint Tracking Node                               │
│     Angle Calc → Dead Zone → Velocity Control → Jog Multi Axis      │
│              제어 주기: 50Hz, ~0.5ms                                 │
└───────────────────────────────┬─────────────────────────────────────┘
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                       Doosan M0609                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## 4.5 핵심 알고리즘

### Extended Kalman Filter (9-state)

**상태 벡터:**
$$
\mathbf{x} = [x, y, z, v_x, v_y, v_z, a_x, a_y, a_z]^T \in \mathbb{R}^9
$$

**상태 전이 행렬:**
$$
F = \begin{bmatrix}
I_3 & \Delta t \cdot I_3 & \frac{\Delta t^2}{2} \cdot I_3 \\
0_3 & I_3 & \Delta t \cdot I_3 \\
0_3 & 0_3 & I_3
\end{bmatrix}
$$

### Joint-space P-Control

$$
\dot{\theta}_{J1} = K_p \cdot e_{horizontal}, \quad \dot{\theta}_{J4} = K_p \cdot e_{vertical}
$$

**파라미터:**
- $K_p = 0.5$, Dead Zone = 2°
- Max Velocity: J1=30°/s, J4=40°/s
- Control Rate: 50Hz
